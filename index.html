FlexBox Grid

The examples of the grid on this page are based on a 12 column grid where $gcw is 84px and $ggw is 22px.

It's worth noting that these values will change from project to project. These values are set in the _variables.scss file.

The grid system only kicks in at the small tablet width which at time of going to press was BPD or 768px

As we take a mobile first approach to our builds this means that each .col is 100% wide stacked on top of each other unless specifically overwritten on a component by component basis which is sometimes the case where the design dictates this break from the set rules.

We use the .grid-row as our wrapper class. This is necessary for the grids to work. The .grid-row width is worked out as width: calc(100% + #{$ggw}); then negatively margined to the left by $ggw. This allows us do something nice and clever with the .col which we'll see below this gist.

A column $gcw has the .col class and a .col__inner inner div. These have no set widths and the row container sorts that shit out. These won't conform to the grid rules (apart from $ggw) but will automatically take up the required space dictated by its content. But luckily they still fill the grid row width left to right. That's magic.

The reason we need the .col__inner is because if you cast your mind way back to a few paragraphs ago when I said we negatively margin the .grid-row to the left by $ggw well we now introduce a padding-left: $ggw to the .col. This gives us our columns which then means we can use .col__inner to apply any background colours or padding or any of the normal styling things we would add to our components. Note: DO NOT STYLE THE .col this is only to be used for the grid structure. Always style .col__inner or any additional markup you require as per the normal component build.

Nested Grid - Equal heights

This is an exapmle of a nested grid. Some of these columns have nested columns. Can you guess which?

The columns are using the .grid-row--equal-height class which makes things equal heights. Imagine that. Equal heights columns. Web 3.0 baby.

This example is also using the .s-nested-col--three rule. This makes the .grid-row .col be 33% wide from BPD to infinity and beyond* (*max width of .grid-row dependant). This .*-nested-col--* allows us to set the behaviour of our .col via the parent .grid-row. Confused yet?

In the nested example we've introduced a s-nested-col--three override in one of the nested .grid-row of s-nested-col--two which splits the nested .col into 50% the width of its parent .col as below.
